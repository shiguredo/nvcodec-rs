# 0008 デコーダーのフレームコピー処理が NV12 固定になっている

## 優先度

P0

## 概要

`SurfaceFormat` は NV12 / P016 / YUV444 / YUV444_16bit / NV16 / P216 の 6 種類を公開 API で受け付けるが、
`src/decode.rs` のフレーム取り出し処理 (Y + UV のサイズ計算とコピー) が NV12 前提で固定されている。

NV12 以外のフォーマットを指定すると、実データ配置とコピー量がズレ、
最悪ドライバ側で不正な読み取りを誘発する。

## 対応方針

現時点で NV12 以外の実用需要がないため、`SurfaceFormat` enum を `Nv12` のみに制限する。
他フォーマットが必要になった時点で、コピー処理の分岐と `DecodedFrame` の拡張を同時に入れる。

## 完了内容

`SurfaceFormat` enum を `Nv12` のみに制限した。
P016 / YUV444 / YUV444_16bit / NV16 / P216 のバリアントと対応する `to_sys()` 分岐を削除した。
